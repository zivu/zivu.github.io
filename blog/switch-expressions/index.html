java enhanced switch statements and switch expressions

Check out the Official documentation

What is the problem with the existing switch statements? Problem #1: they are error-prone. A developer might forget to add a break statement. This will lead to an unexpected result. Try to call this method with “lynca mobile” input.

Enhanced switch statements were introduced.vOnly expressions on the right of -> will be executed. This solved problem #1 mentioned above.

What is the problem with the existing switch statements? Problem #2: variable defined inside one arm has a scope of an entire block. Another arm/case cannot use the same variable name.

Enhanced switch statements solve problem #2 mentioned above. A variable inside one arm has a scope of this arm.

Switch expression example. A default case is required if the enum is not used in a case. For enum, example look at printNameAssociatedWithId(Numbers) method below.

Switch expression can be passed and argument to methods.

Jdk 14 introduced the ‘yield’ statement to return value. ‘yield’ is a restricted identifier and no class can be named that way. Switch expression must either produce value or throw an exception. If you try to comment yield statement compiler will complain.

Switch expression can use a traditional representation with colon label “:”.

Interesting case

The default case is not required in case of the enum (if we cover all cases). If omitted it will be added by the compiler. It is not even recommended to add default case, to throw an exception to indicate unexpected value.
